Going Cycling

How the #cycle method works

April 12, 2015

I've decided to try and explain a method that I've never personally used before. Even reading through the Ruby documentation, I'm trying to think about how this particular method would be useful. The #cycle method iterates through a collection infinitely when nothing is passed in as a parameter. However, when you pass in a integer, the #cycle method iterates through the collection that number of times. Let's say we have an array:

names = ["Alex", "Ben", "John", "Ralph"]

If we say names.cycle(2) {|name| puts name } that would print out the following:

Alex
Ben
John
Ralph
Alex
Ben
John
Ralph

You might wonder what this is useful for. I was thinking the same thing. Let's say you wanted to create a flashcard application for yourself. Almost everyone has used flashcards to quiz themselves at some point in their life. We might use the #cycle method to go through an entire collection of flashcards the number of times that we actually want to quiz ourselves so we can commit all the material to memory. To more properly simulate what this might be like, we can actually store an Enumerator in a variable:

cycle_enum = names.cycle

Now, when we want to move on to the next element in the collection, all we need to do is call the enumerator's #next method:

cycle_enum.next #this would give us "Alex"

Hopefully, this gives you some insight in to how this method works and why it can be useful!
